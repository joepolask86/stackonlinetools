## 5. Functional Requirements

### 5.1 Tool System

* Each tool is registered as a modular component (`OnlineTool` interface).
* Tools can have:

  * **Client-side processing** (default)
  * **Optional server-side processing** (via Edge Function)
* Each tool includes:

  * `metadata` (name, description, icon)
  * `schema` (Zod validation)
  * `component` (React UI)
  * `clientAction` and/or `serverAction`

### 5.2 User Data

* Anonymous users: `localStorage` for preferences, history, and favorites
* Registered users: Supabase-managed user preferences, analytics

### 5.3 Performance

* Code-split by category
* Web workers for heavy computation (e.g., image compression, diff check)
* Static site generation (SSG) for tool landing pages
* <100KB initial JS bundle
